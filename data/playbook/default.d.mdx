# Web Accessibility Guidelines Summary

This document will help you familiarize yourself with the [Web Content Accessibility Guidelines (WCAG) 2.1](https://www.w3.org/TR/WCAG21/) and avoid common accessibility mistakes. This summary simplifies the guidelines. If you need detailed information on how a guideline applies, please refer to the official [WCAG 2.1 guidelines](https://www.w3.org/TR/WCAG21/).

The summary covers Level A and Level AA requirements, which are referenced in [public sector procurement laws in the UK](https://www.gov.uk/guidance/accessibility-requirements-for-public-sector-websites-and-apps), [US public sector procurement laws](https://en.wikipedia.org/wiki/Section_508_Amendment_to_the_Rehabilitation_Act_of_1973), [EU public sector procurement laws](http://mandate376.standards.eu/standard/technical-requirements/#9), and [public sector procurement laws around the world](https://www.w3.org/WAI/policies/).

## 1. Easy to perceive

Your website should present information in ways that are accessible to everyone, regardless of their preferred mode of content consumption (e.g., low vision or screen reader usage).

### 1.1 Text alternatives

  - [1.1.1 Provide text alternatives for non-text content](/playbook/1-1-1-provide-text-alternatives-for-non-text-content)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br />Provide alternative text descriptions for non-text content to convey the same message or functionality.

### 1.2 Provide alternatives for audio content, videos and presentations

  - [1.2.1 Transcript for pre-recorded audio-only content](/playbook/1-2-1-transcript-for-pre-recorded-audio-only-content)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <br />For audio content that has no video (like a podcast), provide a transcript.

  - [1.2.2 Captions for pre-recorded videos](/playbook/1-2-2-captions-for-pre-recorded-videos)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <br />Provide captions for videos that are pre-recorded.

  - [1.2.3 Text or audio description for pre-recorded videos](/playbook/1-2-3-text-or-audio-description-for-pre-recorded-videos)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <br />For videos that are pre-recorded, provide an equivalent text description, or audio description of what happens on screen.

  - [1.2.4 Captions for live videos and presentations](/playbook/1-2-4-captions-for-live-videos-and-presentations)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <br />Provide captions for videos and presentations that are performed live.

  - [1.2.5 Audio description for pre-recorded videos](/playbook/1-2-5-audio-description-for-pre-recorded-videos)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <br />For videos that are pre-recorded, provide audio description of what happens on screen, even if you've already provided a text description.

### 1.3 Create content that can be presented in different ways

- [1.3.1 Information and relationships](/playbook/1-3-1-information-and-relationships)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure that information and relationships that are conveyed visually (like distinct sections within a page, or a label next to a checkbox) are also identified in code.

- [1.3.2 Order of elements in code](/playbook/1-3-2-order-of-elements-in-code)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure that elements appear in a logical reading order in code, so that they are presented in a meaningful order to screen reader users.

- [1.3.3 Instructions don't rely on sensory characteristics](/playbook/1-3-3-instructions-dont-rely-on-sensory-characteristics)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>If you give tips or instructions, do not assume that users can perceive colour, size, shape, sound or the location of elements on screen.

- [1.3.4 Orientation](/playbook/1-3-4-orientation)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure a page's view is not locked to either portrait or landscape mode, unless this is essential.

- [1.3.5 Purpose of text fields](/playbook/1-3-5-purpose-of-text-fields)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>If a text input field collects information about the user, identify its specific purpose in code (for example email, password or given name).

### 1.4 Make content easy for people to see and hear

- [1.4.1 Use of colour](/playbook/1-4-1-use-of-colour)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Do not use colour as the only way to convey any piece of information.

- [1.4.2 Audio control](/playbook/1-4-2-audio-control)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>If any audio content plays automatically for more than three seconds, give people a way to stop it.

- [1.4.3 Text contrast](/playbook/1-4-3-text-contrast)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Make sure that text has enough contrast against the background colour.

- [1.4.4 Resize text](/playbook/1-4-4-resize-text)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure it is possible to complete all tasks when text is resized up to 200%.

- [1.4.5 Images of text](/playbook/1-4-5-images-of-text)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Use real text rather than images of text.

- [1.4.10 Reflow](/playbook/1-4-10-reflow)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure users can access all information and functionality on a screen that's as wide as on the iPhone5, without needing to scroll in both directions.

- [1.4.11 Non-text contrast](/playbook/1-4-11-non-text-contrast)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Make sure that visual information that identifies important graphics, interactive controls and their state has enough contrast against adjacent colours.

- [1.4.12 Text spacing](/playbook/1-4-12-text-spacing)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Ensure that no information or functionality gets lost if users increase the space between lines, paragraphs, letters and words.

- [1.4.13 Extra content on hover or focus](/playbook/1-4-13-extra-content-on-hover-or-focus)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>If content (like a tooltip) appears when users hover over an element with their mouse (or tab to it with their keyboard), it must be easy to dismiss, easy to reach and remain visible.

## 2. Easy to operate

Your website/app must be easy to navigate and use, no matter how someone interacts with it.

For example people who do not use a mouse may use their voice, or press the 'Tab' key to move their keyboard's focus to interactive controls.

### 2.1 Make all functionality work with a keyboard

- [2.1.1 Keyboard](/playbook/2-1-1-keyboard)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure every task can be completed using just a keyboard.

- [2.1.2 No keyboard trap](/playbook/2-1-2-no-keyboard-trap)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure that keyboard-only users do not get trapped within any element.

- [2.1.4 Character key shortcuts](/playbook/2-1-4-character-key-shortcuts)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>If you've added custom keyboard shortcuts, provide a way to switch off or remap these shortcuts.

### 2.2 Give people enough time to read and use content

- [2.2.1 Timing adjustable](/playbook/2-2-1-timing-adjustable)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>If there is any time limit (like a session timeout), give people an easy way to extend it.

- [2.2.2 Visual distractions](/playbook/2-2-2-visual-distractions)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Do not show anything that automatically blinks, scrolls, animates or updates frequently if it lasts more than 5 seconds. Or give people a way to stop it.

### 2.3 Make sure it won't trigger seizures

- [2.3.1 Flashes](/playbook/2-3-1-flashes)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Do not show anything that flashes more than three times a second.

### 2.4 Help people navigate and find content

- [2.4.1 Skip to main content](/playbook/2-4-1-skip-to-main-content)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Give keyboard and screen reader users a way to go directly to the page's main content.

- [2.4.2 Page title](/playbook/2-4-2-page-title)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Give every page a unique and helpful title that indicates the purpose of the page.

- [2.4.3 Focus order](/playbook/2-4-3-Focus-order)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>

  Make sure that interactive controls receive focus in an order that makes sense, when users navigate through them with the keyboard.

- [2.4.4 Link purpose](/playbook/2-4-4-link-purpose)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>

  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Make sure the purpose of every link is clear from the link text alone, or together with associated content (if screen readers recognise the association).

- [2.4.5 Multiple ways to find a page](/playbook/2-4-5-multiple-ways-to-find-a-page)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Give people different ways of finding each page (like searching or browsing links), unless the page is a step in a process.

- [2.4.6 Headings and labels](/playbook/2-4-6-headings-and-labels)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <br/>Headings and form labels should help people find content and complete tasks.

- [2.4.7 Focus visible](/playbook/2-4-7-focus-visible)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>

  <br/>Make sure that keyboard-only users can clearly see which interactive control is focused when they tab through them.

### 2.5 Make it work with other input methods beyond keyboard

### 2.6 Make it work in touchscreens

## 3. Easy to understand

Your website/app must make it easy for people to understand information and how to complete tasks.

### 3.1 Make text easy to understand

- [3.1.1 Language of page](/playbook/3-1-1-language-of-page)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>In code, identify the language that the content of the page is written in (English for example).

- [3.1.2 Language of parts](/playbook/3-1-2-language-of-parts)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level AA</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-blue-400 rounded-md">Content</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>If the page has content in more than one language (for example, if a page in English has a button labelled in Welsh), identify the language of each part in code.

### 3.2 Make things appear and behave in consistent, predictable ways

- [3.2.1 Changes on focus](/playbook/3-2-1-changes-on-focus)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-green-500 rounded-md">Design</span>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black bg-red-400 rounded-md">Code</span>
  <br/>Just navigating to an interactive control with the keyboard shouldn't trigger any action, and shouldn't move the keyboard focus somewhere else.

- [3.2.2 Changes on input](/playbook/3-2-2-changes-on-input)<br/>
  <span className="inline-flex items-center px-2 py-1 text-sm font-medium text-black rounded-md bg-stone-300 my-1">Level A</span>
